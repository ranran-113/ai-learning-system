## ⚠️ 重要变化

v2.0 是对 v0.1-v0.3 的**框架级重做**，**不是叠加**。

**v0.x 时代**做对的事（保留）：
- 0-10 级 AI 能力地图
- 三导师系统（卡帕西 / 钱学森 / 阿德勒）
- 苏格拉底式教学
- 联合测试 + 学习人格分类
- 输出沉淀 / 原子笔记
- AI 通识 16 章 + AIPM 16 章教材（**v0.3.0 完成**）

**v0.x 时代**做错的事（v2.0 推翻）：
- ❌ 把 12 节课当主线 → **v2.0：32 章教材是唯一系统化内容，12 节课作废**
- ❌ chat-first 占首页 C 位 → **v2.0：学习路径占首页 C 位，chat 是某一节的深化模式**
- ❌ 侧边栏 8 个平铺项 → **v2.0：4 条学习主线 + 持续学习 + 资产 + 设置**
- ❌ 缺 AI 工具线（Claude Code 等）→ **v2.0：补**
- ❌ 缺 AIPM 求职线（简历 / 面试 / 入职）→ **v2.0：补**
- ❌ 缺 GitHub 周榜 / 博客 / 访谈源 → **v2.0：补**
- ❌ 沉淀没出口（Obsidian / 自媒体）→ **v2.0：建出口**

具体迁移路径见 `ROADMAP.md`。

## 1. 产品定位

本产品是一个 **AI 学习成长系统**。它帮助不同基础的用户**持续学习 AI、把学到的东西沉淀成自己的资产、形成良性学习循环**。

核心主线：

```
持续学习 AI  →  沉淀成资产  →  能力成长  →  反哺持续学习
```

产品**不是**：
- 普通课程平台（不是 ▶️ 看视频打卡）
- 普通 AI Chat（不是 ChatGPT 套壳）
- AI 资讯聚合站（不是只刷热点）
- 工具型助手（不是只查 prompt）

产品**真正解决的问题**：

> **AI 知识爆炸时代，用户无法系统学、持续学、沉淀不下来、形不成良性循环。**

---

## 2. 产品一句话

> 一个面向 AI 学习者和 AIPM 的**多路径**持续成长系统，通过 **0-10 能力地图 + 苏格拉底教学 + 三导师陪伴 + 4 大学习线 + 持续学习区 + 资产沉淀出口**，让你不只是「会用 AI」，而是真正**长成一个 AI 时代有判断力的人**。

更口语：

> 不是让你收藏更多 AI 知识，是陪你**真正学懂、说出来、用起来、沉淀下来，形成自己的方法论**。

## 3. 目标用户

### 3.1 AI 小白
- 痛点：想学但不知道从哪开始 / 看到工具论文就焦虑 / 不知道自己在哪
- 需要：**清晰的起点 + 低压力节奏 + 看懂的概念解释**
- 入口：**测试 → 推荐 AI 通识第一章**

### 3.2 AI 爱好者 / 自学者
- 痛点：看很多热点 / 收藏一堆 / 学得散 / 没体系 / 学完不输出
- 需要：**能力进阶地图 + 系统化课程 + 输出沉淀 + 资讯结构化**
- 入口：**测试 → 看自己 Lv.X → 选学习线 + 接入持续学习区**

### 3.3 AIPM / 想转 AIPM
- 痛点：要理解技术 + 产品 + 商业 / 不知技术要学到什么程度 / 不知怎么从知识走向产品
- 需要：**AIPM 主线 + 求职专项 + Agent / RAG 等技术产品化讲解**
- 入口：**测试 → AIPM 学习线 + AIPM 求职子线**

### 3.4 AI 从业者 / 进阶用户
- 痛点：已经会用 / 需要持续跟新论文新工具 / 需要把新知识转成方法论
- 需要：**持续学习区（热点 / GitHub / 博客 / 论文）+ 跳过新手区**
- 入口：**测试 → 系统识别为 Lv.7+ → 直接进持续学习区 + 高阶教材章节**

### 3.5 高焦虑 / 易内耗学习者
- 痛点：信息过载 / 自我怀疑 / 启动困难 / 一断更就放弃
- 需要：**情绪接住 + 任务缩到 5 分钟 + 不被等级羞辱**
- 入口：**测试识别情绪信号 → 阿德勒导师介入 → 缩任务**

### 3.6 想做 AI 内容输出的人（新增）
- 痛点：学的东西放在脑里 / 想做自媒体但没素材 / 想搭个人方法论
- 需要：**学习 → 笔记 → 自媒体草稿 / 导出 Obsidian 一键链路**
- 入口：**资产中心 → 沉淀转输出**

## 4. 核心信念（不变）

无论框架怎么变，以下 5 条永远是产品红线：

1. **AI 学习是主线**：所有功能围绕「**真正学懂 AI**」，不是娱乐化 / 资讯化
2. **沉淀是闭环**：每次学习必须导向一个可保存的输出
3. **低压力成长**：不羞辱、不焦虑化、不做等级鄙视链
4. **苏格拉底优先**：导师不直接灌输，通过问题引导用户自己想通（除阿德勒外）
5. **三导师明确分工**：卡帕西 = 知识 / 钱学森 = 决策 / 阿德勒 = 情绪。**这是产品身份的一部分**

## 5. 产品框架（v2.0 核心）

### 5.1 三层结构

```
┌─────────────────────────────────────────────────────────┐
│                  第 1 层：4 大学习线                      │
│  系统化的内容主线 —— 走完一条线 = 完整掌握一个方向         │
├─────────────────────────────────────────────────────────┤
│              第 2 层：持续学习区（横切）                  │
│  保持 AI 圈感知 —— 热点 / GitHub / 博客 / 论文            │
├─────────────────────────────────────────────────────────┤
│                第 3 层：资产沉淀（出口）                   │
│  把学到的变成自己的 —— 原子笔记 / Obsidian / 自媒体        │
└─────────────────────────────────────────────────────────┘
```

**三层互相支撑**：
- 学习线产生**有结构的知识**
- 持续学习区产生**前沿的信号**
- 资产沉淀让两者都**长成自己的东西**

### 5.2 4 大学习线

```
┌──────────┬──────────┬──────────┬──────────┐
│ AI 通识   │  AIPM   │ AI 工具  │ AIPM 求职 │
│          │          │          │           │
│ 小白入口  │ 转岗入口 │ 工具入口 │ 求职入口   │
│          │          │          │           │
│ 教材 16 │ 教材 16 │ Claude  │ 简历 + 面试│
│ 章       │ 章       │ Code 等  │ + 入职清单 │
└──────────┴──────────┴──────────┴──────────┘
```

每条线都有：**进度 + 推荐下一步 + 四种动作（读 / 聊 / 沉淀 / 费曼）**。

> **关于「四动作」(v2.0 升级)**：v0.x 时代是「读 / 聊 / 沉淀」三动作。v2.0 加入**费曼挑战**作为第 4 个动作 —— 用「能用一个 10 岁孩子能听懂的话讲清这件事」检验是否真懂。详见 §6.0。

### 5.3 持续学习区

```
┌────────────────────────────────────────┐
│  🔥 AI 热点          → 每日精选        │
│  📊 GitHub 周榜      → 每周自动抓      │
│  📝 博客 / 访谈精选   → 策展精选        │
│  📄 论文导读          → 已有 15 篇      │
└────────────────────────────────────────┘
```

**关键设计**：这一区**横切** 4 大学习线 —— Lv.7 用户可以**只用持续学习区**，不走教材；Lv.2 用户**先把通识线学一段**再接入。

### 5.4 资产沉淀

```
┌────────────────────────────────────────┐
│  💎 原子笔记 → 一句话 / 短笔记 / 长输出 │
│  ↓                                       │
│  ├─ 导出 Markdown 到 Obsidian / Notion  │
│  ├─ 转自媒体草稿(微信 / 小红书 / X)     │
│  └─ 作为简历 / 作品集证据(求职用)        │
└────────────────────────────────────────┘
```

**这一层是产品和用户长期价值绑定的钥匙**。用户的笔记跟着他走，**不锁在我们系统里**。

## 6. 4 大学习线详解

### 6.0 每章 3 主动作（UI 上）/ 4 子状态（底层）

每一章（无论 AI 通识 / AIPM / AI 工具 / AIPM 求职）UI 上有 **3 个主动作**：

| 动作 | 干什么 | 完成判定 | 状态 |
|---|---|---|---|
| 📖 **读** | 看章节正文 | 进过阅读页 | v0.3.0 已实现 |
| 💬 **聊&沉淀** | 和导师对话深化 + 在对话中沉淀成原子笔记（沉淀按钮在 chat 内） | **聊过 AND 沉淀过** —— 只聊不沉淀 = 半完成（⚠ 提示） | v0.3.0 已实现 |
| 🎯 **费曼** | 用 10 岁孩子能听懂的话讲清这一章，AI 扮演「孩子」反问 + 评价 | 做过挑战(passed/almost) | **v0.4+ 新增** |

**重要的设计原则**（v0.4.4 修正）:

> **沉淀不是独立按钮，它是「聊」的子动作**。
>
> v0.4.2 早期错误地把沉淀做成和聊并列的独立按钮，导致两者点击效果一致(都跳 chat)且用户困惑。v0.4.4 修正:UI 合并为「💬 聊&沉淀」一个按钮,沉淀在 chat 内通过「完成沉淀」按钮触发(已实现)。

**底层依然记 4 个状态**（read / chat / note / feynman），用于：
- 「**真懂这一章**」标准：3 主动作全部完成 = 4 子状态全部完成
- 「**聊&沉淀**」按钮的半完成判断（聊过但没沉淀显示 ⚠）

**费曼挑战的工作流**：

```
1. 用户进入「费曼挑战」
2. 屏幕显示:「假装你在跟一个 10 岁孩子讲话。
              用 30 秒说清:[本章核心问题]」
   + 禁用术语列表(本章关键词,强制用大白话)
3. 用户用语音 / 文字输入解释(支持 30-60 秒)
4. AI 扮演「10 岁孩子」反问 1-2 个最常见的「为什么 / 我不懂」
5. 用户回答
6. AI 根据用户的表现给「通过 / 还差点 / 没过」评价
   - 通过 → 本章四动作完成,这一章「真懂了」
   - 还差点 → 指出卡在哪个概念,推荐回去看本章第 X 段
   - 没过 → 鼓励性反馈,推荐先沉淀一条笔记再重试
```

**费曼挑战的产品价值**：

1. **检验真伪懂**：读完和真懂之间隔了一层。费曼是最锋利的检验
2. **沉淀的高质量素材**：用户讲清楚的版本本身就是好的原子笔记 / 自媒体素材
3. **触发主动复习**：卡壳的地方反推回章节，形成「漏点闭环」
4. **训练「能教会别人」的能力**：这是 AIPM 跨职能沟通的核心技能

**注意：费曼挑战不是第 4 个导师**。AI 扮演的「10 岁孩子」是 prompt 模式，不出现在三导师卡片里。**保持产品身份是「三导师」**，不稀释。

详细页面规格见 §12.6。

## 7. 持续学习区（横切 4 大线）

### 7.0 持续学习区 ↔ 教材的双向链接（v2.0 新增）

**问题**：教材是「**稳态结构化知识**」，持续学习区是「**前沿动态信号**」。如果让前者直接被后者改写，会失去稳定性。

**v2.0 的设计**：**不改正文，但建桥**。

**桥的形式 1：教材每章下方「📡 这一章在现实中」区**

每个教材章节页底部自动聚合：
```
教材 AI 通识 C6 RAG
─────────────────────
[正文,稳定不变]

──────────────
📡 这一章在现实中

🔥 [热点]：Perplexity 2025 Q1 API 大降价(2 天前)
📊 [GitHub]：langchain-ai/langchain(本周 stars +1.2K)
📄 [论文]：RAG 综述 2024(近 3 个月)
📝 [博客]：「RAG is dead, long live agentic retrieval」

[让导师讲一下:这些动态怎么影响你之前学的]
```

**桥的形式 2：持续学习区 → 关联教材章节**

每条持续学习内容（热点 / GitHub / 博客 / 论文）显示「**对应教材：[章节链接]**」。用户从最新动态反向找到系统化基础。

**实现机制**：

- **聚合方式**：tag 匹配 + 关键词匹配（每章带 `relatedTopics: string[]`，持续学习内容带 `tags: string[]`，匹配显示）
- **新鲜度**：仅展示近 90 天内容
- **数量**：每章最多 5 条相关动态
- **MVP 阶段**：Step 3（v0.6.x）实现

**不在 v1.0 范围**：
- 教材正文的季度增量更新（v1.0 后人工 review）
- 用户沉淀反推教材改进 backlog（v2.0+）

### 7.1 AI 热点（已有，v0.5 升级）

**升级方向**：

- 沉淀按钮 → 直接生成原子笔记
- 关联到 4 大线的相关章节（「**这条热点对应教材 X 章**」）

### 7.2 GitHub 周榜（v0.6 新做）

**MVP 设计**：
- 每周一自动抓 GitHub trending（AI 相关，按 stars + 增量排序）
- 取前 5 个项目
- 每个项目自动生成「**这周值得看**」摘要（200 字）
- 用户可以点「**让导师讲解**」深化

**例子**：
```
本周第 1: openai/swarm (Multi-Agent 框架)
   star 增量: 3.2K  ·  Python  ·  Apache-2.0

   为什么值得看:
   OpenAI 官方发布的轻量 Multi-Agent 框架。它对外宣称
   不进 production,但工程范式值得学。

   [让卡帕西拆解] [对应教材 AI C9] [收藏]
```

## 8. 资产沉淀（出口闭环）

### 8.1 原子笔记（已有，v0.4 重组）

继续 v0.1.6 的原子笔记三原则：
- 一笔记一问题
- 用自己的话
- 脱离原文也能看懂

**v0.4 改动**：从「**沉淀**」侧边栏入口 → 「**资产**」一级模块。

**笔记类型（v2.0 扩展）**：

| 类型 | 描述 | 来源 |
|---|---|---|
| `one_sentence` | 一句话总结 | 沉淀 |
| `note` | 短笔记（100-500 字） | 沉淀 |
| `prompt` | Prompt 模板 | 沉淀 |
| `social_post` | 自媒体草稿 | 沉淀转出 |
| `speech_summary` | 口语版总结 | 沉淀 |
| **`feynman`** | **费曼挑战记录（用户讲解 + AI 反问 + 评价）** | **v0.4+ 新增** |

**费曼笔记的字段**（在 `OutputRecord` 上扩展）：
```ts
{
  type: "feynman",
  content: string,           // 用户的讲解
  feynman: {
    chapterId: string,        // 哪一章的费曼
    userExplanation: string,  // 用户讲解
    childQuestions: string[], // AI 反问
    userFollowUps: string[],  // 用户跟进回答
    result: "passed" | "almost" | "needs_work",
    aiReview: string,         // AI 评价
    durationSeconds: number,
  }
}
```

费曼笔记可以转自媒体、可以导出 Markdown、可以作为简历证据（详见 §8.3 / §8.4）。

## 10. 三导师（产品身份的一部分）

### 10.1 卡帕西（知识深度）

- 定位：AI 技术深度与理解导师
- 方法：苏格拉底式提问 + 卡帕西式拆穿黑箱
- 长度：≤ 80 字（铺垫可到 120）
- 出现：AI 通识技术章节 / 用户问技术细节 / 用户卡在概念

### 10.2 钱学森（系统决策）

- 定位：系统工程 + 决策思维导师
- 方法：苏格拉底式提问 + 系统拆解 + 反向思考 / 能力圈 / 机会成本（吸纳芒格工具）
- 长度：≤ 150 字
- 出现：AIPM 章节 / 用户做决策 / 系统设计 / 模拟面试主导

### 10.3 阿德勒（情绪陪伴）

- 定位：成长陪伴 + 情绪重构导师
- 方法：罗杰斯式开场 + 阿德勒哲学内核（**不使用苏格拉底**）
- 长度：60-100 字，多短句
- 出现：用户情绪低落 / 断更回归 / 输出恐惧 / 自责

### 10.4 在 v2.0 产品中的位置（变化）

**v0.x 错误做法**：chat 占首页 C 位 → 用户感觉「**这是个 chat 工具**」。

**v2.0 正确位置**：
- 三导师**不占首页 C 位**，首页是学习路径
- 用户进某一章 / 某个任务 → 才进 chat 模式 → 三导师出现
- 测试结果页明确告知「**你的导师组合**」（保留 v0.x 的设计）
- 介绍页 / About 页强调「**这是三导师 AI 学习系统**」（品牌身份）

### 10.5 切换规则（不变）

由系统路由节点根据用户状态、学习内容、情绪信号自动切换。导师本身不谈论切换。详见 MENTORS.md。

- 通过 / 还差点 / 没过三档结果
- 通过率作为「**这一章真懂率**」指标
- 费曼记录存为 `OutputRecord` 类型 `feynman`，可导出 / 转自媒体

## 13. MVP 边界（v2.0）

### MVP 必须做

- 4 大学习线的导航 + 入口（**Step 1**）
- AI 通识 + AIPM 教材内容已就绪
- 三导师 + 苏格拉底 + 0-10 级（已有）

### MVP 不做

- 真实的自媒体平台发布（**只生成草稿，不直接发**）
- AI 工具线的视频教程（**先用文字 + 截图**）
- 复杂的等级提升动效（**第一版静态显示**）

### Phase 2 候选

- 语音 / 视频导师
- 社区
- 课程销售
- 移动端 native

## 14. 失败兜底

任何技术失败都要有兜底（不变）：
- AI 调用失败 → 提示重试 + 保留输入
- 语音识别失败 → 回退文字
- 热点接口失败 → 本地示例
- 保存失败 → 提示下载本地

新增（v2.0）：
- 学习线进度数据丢失 → 从已沉淀的笔记反推
- 测试结果丢失 → 显眼提示重测，不强制

## 16. 改动追踪原则

从 v2.0 开始，**任何改动都要回答**：

```
1. 这件事属于 4 大学习线 / 持续学习 / 资产沉淀 / 系统层 哪一块?
2. 涉及的用户路径是什么?
3. 改完之后用户能在哪个具体页面看到差别?
4. 有没有让某条主线更清晰 还是又把它弄乱了?
```

**没有回答这 4 个问题的改动不动手**。这是对 v0.x 「**叠加式迭代**」的纠偏。
