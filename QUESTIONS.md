## 2. 第一部分：学习人格画像（Q1-Q5）

### Q1（单选）你最近一周学 AI 的状态最像哪个？

| 选项 | 维度影响 |
|------|---------|
| A. 我每天都有学一会，状态稳定 | persistence +5, anxiety -2 |
| B. 我想学但开了几次后就停了 | persistence -3, selfBlame +3 |
| C. 我学得很猛但只持续了几天就累了 | persistence -2, anxiety +3 |
| D. 我看了很多收藏，但真正学进去的少 | persistence -2, anxiety +4 |
| E. 我最近完全没学，状态不太好 | persistence -5, selfBlame +5 |

**设计意图**：直接探"持续性"卡点。E 是断更 + 自责双信号。

### Q2（单选）看到一个新 AI 概念你不懂时，通常会怎样？

| 选项 | 维度影响 |
|------|---------|
| A. 觉得有意思，立刻去查 | anxiety -3, persistence +2 |
| B. 觉得焦虑，担心又是新东西要追 | anxiety +5 |
| C. 收藏起来，告诉自己晚点看（然后大概率没看） | persistence -3, anxiety +2 |
| D. 想搞懂，但被术语劝退 | anxiety +3, selfBlame +2 |
| E. 跳过，反正不影响我现在用 | anxiety -1, persistence -1 |

**设计意图**：检测信息过载焦虑。B / C 是信息过载型核心信号。

### Q3（单选）如果让你现在把今天学的写一句话发出来，你会？

| 选项 | 维度影响 |
|------|---------|
| A. 直接发，有什么发什么 | outputWillingness +5 |
| B. 想发，但会反复改几遍才发 | outputWillingness +2, selfBlame +3 |
| C. 想发，但担心写得不专业，最后不发 | outputWillingness -3, selfBlame +4 |
| D. 不太想发，觉得没必要让别人看 | outputWillingness -3 |
| E. 我会发，但只发给特定的人看 | outputWillingness +2 |

**设计意图**：直接探输出恐惧。C 是完美主义自责型核心信号。

### Q4（单选）你更愿意接受哪种学习节奏？

| 选项 | pacePreference |
|------|---------------|
| A. 每天 10-15 分钟，稳定一年 | `daily_micro` |
| B. 一周 2-3 次，每次 30-60 分钟 | `weekly_medium` |
| C. 周末集中学，工作日不学 | `weekend_burst` |
| D. 兴趣来了猛学，没兴趣就停 | `interest_driven` |
| E. 我不确定，看推荐 | `uncertain`（默认 `daily_micro`） |

**设计意图**：决定 paceRecommendation。E 是重要兜底，不强迫用户立刻规划自己。

### Q5（单选）学一件事中途卡住，你的第一反应是？

| 选项 | 维度影响 |
|------|---------|
| A. 觉得是自己不够聪明 | selfBlame +5 |
| B. 觉得是方法不对，换个角度 | selfBlame -3, persistence +2 |
| C. 想找人问，但又怕暴露自己不懂 | selfBlame +3, outputWillingness -2 |
| D. 觉得算了，可能不适合我 | selfBlame +2, persistence -3 |

## 3. 第二部分：AI 能力等级行为指纹（Q6-Q12）

指标格式：`[Lv.X: 权重]`。权重 1 / 3 / 5 表示弱 / 中 / 强信号。

### Q6（多选）下面哪些是你**最近 30 天真做过**的？

| 选项 | 指标 |
|------|------|
| A. 没怎么用过 AI | `Lv.0: 5` |
| B. 让 AI 写邮件 / 总结文章 / 查资料 | `Lv.1: 3` |
| C. 我会追问、补充背景信息给 AI | `Lv.2: 3` |
| D. 我写过带格式/语气/范围要求的多段 Prompt | `Lv.3: 3` |
| E. 同一周内用 AI 做过 ≥ 3 类完全不同的事 | `Lv.4: 3` |
| F. 我有自己保存的 Prompt 模板或资料库 | `Lv.5: 3` |
| G. 我用过 Claude Code / Cursor / Cline 等 AI 编程工具 | `Lv.6: 3` |
| H. 我写过 Agent / Skill / 项目规则 / 自定义自动化 | `Lv.7: 5` |
| I. 我用 AI 做出过真实的产品 / 工具 / 作品 | `Lv.8: 5` |
| J. AI 已默认介入我每天的工作流 | `Lv.9: 3` |
| K. 我有自己的 AI 方法论 + 工具链 + 知识库体系 | `Lv.10: 5` |

**设计意图**：核心证据题。允许 0-11 项多选，全勾或全不勾都有效。

### Q7（单选）想象你要让 AI 帮你写一份产品介绍，你最像哪种做法？

| 选项 | 指标 |
|------|------|
| A. 直接说"帮我写个产品介绍" | `Lv.1: 3` |
| B. 加上"我们的产品是 XXX，目标用户是 XXX" | `Lv.2: 3` |
| C. 还会要求格式、长度、语气、举例风格 | `Lv.3: 3`, `Lv.2: 1` |
| D. 我有自己的 Prompt 框架/模板，套着写 | `Lv.5: 3`, `Lv.3: 1` |
| E. 我会让 AI 先反问我，基于答案再生成 | `Lv.4: 3`, `Lv.3: 2` |

**设计意图**：Lv.1-3 分水岭。D 比 E 等级高，因为 D 是沉淀的模板（Lv.5），E 是元提示技巧（Lv.4）。

## 6. UX 注意事项

1. **进入前一句话定调子**：
   > "这 15 道题不是考试，是帮我们了解你 —— 等级也不是评判，是地图上的位置。"

2. **每屏 1 题**，顶部进度条 `3/15`，下方"上一题/下一题"。

3. **Q6 和 Q9 是多选**，UI 要明确标"多选，可选多个"。

4. **答题时长 < 60 秒** → 后端标记 `suspiciousAnswer = true` + UI 弹一次温和确认："这速度有点快，要再看看吗？"（一次，不强制）。

5. **冲突检测温和提示**（当答案出现矛盾，如 Q6 同时勾 A 和 K）：
   ```
   我们注意到你既选了「没怎么用过 AI」也选了「有完整 AI 方法论」——
   这两个看起来不太一样。
   想跟你确认一下：哪一个更像你现在的状态？
   [更接近 A] [更接近 K] [都对，继续保留]
   ```
   第三个选项永远开放，不强制改答。保留时设 `evidenceConflict = true`，首次会话由阿德勒温和回访（见 `TECH.md` §8.3 路由优先级 0）。

6. **结果页布局**（治愈风）：
   ```text
   你是「信息过载型」
   
   你的位置在 Lv.3 「AI 控制者」
   （置信度 70%，可能在 Lv.2-Lv.4 之间）
   
   下一站：Lv.4 「AI 跨场景使用者」
   要解锁：开始在 ≥ 3 类不同任务里用 AI
   
   推荐节奏：每天 10 分钟，稳定一个月
   今天的第一步（5 分钟以内）：[具体动作]
   
   觉得算高了 [-1] / 算低了 [+1]   （只能微调一次）
   
   [开始第一节学习]
   ```

7. **微调按钮**：只能调一次，存为 `ai_level_user_adjusted`。原始算法结果保留为 `ai_level_algorithm`。当前生效 `ai_level = user_adjusted ?? algorithm`。
