# CLAUDE.md - Claude Code 项目规则

本文件是 Claude Code / AI Coding 工具在本项目中的项目级规则。任何自动化代码生成、重构、后端开发、前端开发、文档更新都必须遵守本文件。

**当前框架版本：v2.0**（详见 `PRD.md` v2.0 + `ROADMAP.md`）。v0.4.0 起按 v2.0 框架推进。

---

## 1. 项目目标

本项目是一个 **AI 学习成长系统**（v2.0 起去掉"三导师"作为唯一定语，但三导师依然是产品身份的一部分）。

核心目标：

```text
持续学习 AI → 沉淀成资产 → 能力成长 → 反哺持续学习
```

系统帮助**不同基础的用户**（小白 / 爱好者 / AIPM / 求职者 / 进阶用户 / 内容输出者）通过 **4 大学习线 + 持续学习区 + 资产沉淀出口** 持续学习 AI。

本项目**不是**：
- 普通课程平台
- 普通 AI Chat
- AI 资讯聚合站
- 工具型助手

---

## 2. 产品核心原则（v2.0 更新）

任何开发都不得违背以下原则：

1. **AI 学习是主线**：所有功能围绕 AI 学习自然生长，**不能把自媒体输出 / 英语 / 表达 等变成并列主线**。
2. **沉淀闭环**：每次学习必须导向可保存的输出，**且要有出口到 Obsidian / 自媒体 / 简历**。
3. **低压力成长**：不羞辱、不焦虑化、不做等级鄙视链。
4. **苏格拉底优先**：卡帕西 + 钱学森用苏格拉底式提问引导，阿德勒用罗杰斯式开场 + 阿德勒哲学内核（不用苏格拉底）。
5. **三导师明确分工**：卡帕西 = 知识 / 钱学森 = 决策 / 阿德勒 = 情绪。**这是产品身份的一部分，但不占首页 C 位**。
6. **产品决策必须询问负责人**：定位 / 功能增删 / 导师设定 / 学习线结构 / 视觉风格等。
7. **教材 32 章是唯一系统化内容**：v0.3.0 已完成。12 节课作废（`LESSONS.md` 标 deprecated，仅作历史参考）。
8. **4 大学习线是产品骨架**：AI 通识 / AIPM / AI 工具 / AIPM 求职。**任何新功能都要回答「属于哪条线」**。
9. **0-10 级和教材绑定**：每章标 `targetLevelMin / Max`，系统根据用户等级智能推荐。
10. **任何改动前问 4 个问题（PRD §16）**：
    - 这件事属于 4 大学习线 / 持续学习 / 资产沉淀 / 系统层 哪一块？
    - 涉及的用户路径是什么？
    - 改完之后用户能在哪个具体页面看到差别？
    - 有没有让某条主线更清晰、还是又把它弄乱了？

    **答不出来的改动不动手**。这是对 v0.x「叠加式迭代」的纠偏。

## 4. 禁止行为

未经项目负责人确认，禁止：

1. 擅自改变产品定位（4 大学习线 + 持续学习 + 资产沉淀的三层结构）。
2. 擅自删除三导师系统（卡帕西 / 钱学森 / 阿德勒）。
3. 擅自删除 AI 能力 0-10 级进阶地图。
4. 擅自把产品改成普通 ChatBot。
5. 擅自把产品改成普通课程平台。
6. 擅自把 AI 热点学习舱改成资讯站。
7. 擅自把 MVP 加入支付、社区、复杂视频、实时语音等重功能。
8. 擅自引入大型复杂技术栈。
9. 擅自改数据库核心结构。
10. 擅自改变登录策略。
11. 擅自改变导师人设、语气或教学方式。
12. 擅自删除文档或核心目录。
13. 擅自删除 `lib/textbooks/ai/` 或 `lib/textbooks/aipm/` 下的 32 章教材内容（这是 v0.3.0 完成的核心资产）。
14. 擅自删除 `QUESTIONS.md` / `MENTORS.md` 中的核心内容（4 种画像类型、三导师 system prompt、路由规则等）。
15. **v2.0 新增**：擅自违反「**任何改动前 4 个问题**」原则（见 §2.10）。叠加式迭代是错误，必须先回到 PRD v2.0 框架确认改动归属。
16. **v2.0 新增**：擅自让 chat 占首页 C 位 —— v2.0 首页是学习路径，chat 是某一节的深化模式。
17. **v2.0 新增**：擅自把 12 节课重新拉回主流程 —— 已 deprecated，仅作历史参考。

## 5. 产品决策规则

遇到以下情况，必须先询问项目负责人：

- 功能边界不清楚
- 课程内容不明确
- 交互方式有多个方案
- 页面视觉风格需要选择
- 数据结构影响后续扩展
- 需要增加新页面
- 需要删除或合并页面
- 需要新增第三方服务
- 需要改变用户路径
- 需要调整 AI 能力等级定义
- 需要改变三导师出场逻辑

默认行为：**不擅自决策，先提出方案，让项目负责人选择。**

## 7. 代码风格规则

1. 使用 TypeScript。
2. React 组件使用函数式组件。
3. UI 使用 Tailwind CSS 和 shadcn/ui。
4. 页面结构保持清晰，避免一个文件过大。
5. 业务逻辑不要写死在组件里，应放到 `lib/` 或 API route 中。
6. Prompt 和导师规则放到 `lib/prompts/`。
7. LangGraph 工作流放到 `lib/langgraph/`。
8. 数据库访问封装到 `lib/supabase/`。
9. 所有复杂函数需要写简短注释。
10. 不要随便加入复杂状态管理库，MVP 优先简单实现。

## 9. 三导师实现规则

三导师由系统路由节点（`lib/langgraph/routers.ts`）根据用户状态、学习内容、情绪信号**自动切换**，导师本人不负责切换判断，也不在对话中谈论切换。用户通过 UI 对话气泡上的导师名字/头像感知切换。

三位共享同一份 LearningContext（用户画像、完整对话历史、全部历史输出沉淀、学习来源内容、上一位导师交接信息），切换不丢上下文。

苏格拉底式提问是「卡帕西」和「钱学森」共用的教学方法，不是某位导师的专属。阿德勒使用「罗杰斯式开场 + 阿德勒哲学内核」的混合模式，不使用苏格拉底式提问。

### 9.1 卡帕西导师

定位：AI 技术深度与理解导师（人格原型为 Andrej Karpathy）。

职责：

- AI 技术深度（大模型、Prompt、RAG、Agent、LangGraph、上下文、记忆、工具调用、工作流）
- AI 论文导读
- AI 热点中的硬技术内容
- 把复杂概念拆穿、去神秘化
- 通过"从零搭最小可跑版本"帮用户真正理解

教学方法：**苏格拉底式提问 + 卡帕西式拆穿黑箱**

典型行为：

```text
直接问一个能暴露关键的问题
→ 等用户回答
→ 用"所谓 X 其实就是 Y"祛魅
→ 用最小可跑版本或类比说明
→ 要求用户用自己的话总结
```

发言长度：默认 80 字以内；铺垫反例或类比时可放宽到 120 字。

代码命名：`karpathy`（在世真人，公开上线前确认是否更名）。

### 9.2 钱学森导师

定位：系统工程与项目导师。

职责：

- 系统工程
- 目标拆解
- 技术路径
- AIPM 能力
- Agent / 工作流 / 产品架构理解
- **决策思维工具**（吸纳自查理·芒格）：反向思考、能力圈判断、机会成本、避免认知偏误

教学方法：**苏格拉底式提问 + 钱学森式系统拆解**

典型行为：

```text
先确认目标
→ 把问题拆成系统/模块/路径/任务（不超过 4 项）
→ 给二选一或多维度对比
→ 必要时反问"什么会让这件事失败"（反向思考）或"这在你的能力圈里吗"
→ 落到"你下一步具体做什么"（24 小时内可完成）
```

发言长度：≤ 150 字，允许列结构。

代码命名：`qian`

### 9.3 阿德勒导师

定位：成长陪伴与情绪重构导师（人格内核为阿德勒哲学，开场风格为罗杰斯式无条件接纳）。

职责：

- 接住情绪（罗杰斯式开场）
- 重构自责（阿德勒目的论：不是因为差，是选择了不做）
- 课题分离（你的课题是写出来，别人怎么看是别人的课题）
- 断更召回
- 输出恐惧缓解
- 鼓励行动勇气

教学方法：**罗杰斯式开场 + 阿德勒哲学内核**（不使用苏格拉底式提问）

典型行为：

```text
第一句无条件接纳（罗杰斯式）：「听起来你这两天挺累的」
→ 等用户松一下
→ 温和重构（阿德勒式）：「这次停下来，对你来说意味着什么？」
→ 课题分离：「你的课题不是写好，是写出来」
→ 把任务缩到 5 分钟以内
→ 状态稳定后系统路由到知识导师
```

发言长度：60-100 字，多用短句和留白。

代码命名：`adler`

边界：允许用户"今天不学"，但**不纵容长期回避**。同一任务被回避 ≥ 2 次时，阿德勒会温和揭示目的论："你不是不能，是你在选择不做。我们看看为什么。"

## 10. LangGraph 规则

LangGraph 负责工作流编排，不负责所有产品逻辑。

MVP 工作流必须包含：

```text
读取用户画像
→ 读取 AI 能力等级
→ 读取历史输出沉淀（构成全量上下文一部分）
→ 判断学习来源：内置课程 / 上传资料 / AI 热点
→ 路由节点选择导师（基于用户状态、学习内容、情绪信号；连续性 ≥ 2 轮，阿德勒例外）
→ 生成苏格拉底式问题（卡帕西/钱学森）或共情重构回应（阿德勒）
→ 等待用户回答
→ 评估理解程度
→ 追问 / 提示 / 总结
→ 要求输出
→ 保存学习记录
```

## 12. MVP 边界

MVP 不做：

- 视频导师
- 数字人
- 实时语音聊天
- 三位导师专属音色
- 支付系统
- 社区系统
- 完整课程商城
- 扫描 PDF OCR
- EPUB / MOBI / PPT 深度解析
- AI HOT 全量动态和复杂推荐

MVP 必须做：

- 联合测试
- AI 能力 0-10 级地图
- 三导师文字学习
- 语音输入与回复播放
- 内置课程结构
- 上传资料自动拆课
- AI 热点学习舱精选热点
- 成长档案
- 输出记录
- 邮箱保存登录

## 13. 兜底原则

如果某个功能无法稳定实现，优先保证主流程可用。

优先级：

```text
学习闭环 > 数据保存 > 语音体验 > 热点丰富度 > 视觉精致度
```

任何技术失败都要有兜底提示，例如：

- AI 调用失败：提示稍后重试，并保留用户输入。
- 语音识别失败：回退到文字输入。
- 热点接口失败：显示本地示例热点。
- PDF 解析失败：提示用户复制文本或上传 txt/md。
- 保存失败：提示用户下载本地记录。
