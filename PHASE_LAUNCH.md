## 2. 隐私政策 + 用户协议

**必须有**：

- 隐私政策（Privacy Policy）：说明收集哪些数据（邮箱、对话历史、输出沉淀）、保存多久、第三方共享情况（DeepSeek / Supabase / Vercel）、用户如何删除数据
- 用户服务协议（Terms of Service）：使用范围、免责声明（特别是 AI 生成内容免责）、争议解决

**为什么必须有**：

- 收集邮箱 = 收集 PII（个人身份信息），中国法律 + GDPR 都要求明示
- 用户上传 PDF 资料 → 第三方处理 → 必须告知

**决策动作**：找模板（用 termly.io 或 termsfeed.com 这类工具生成基础版），由懂法务的朋友过一眼。上线前 1 周完成。

## 3. API 密钥与 Rate Limit 防护

**问题**：

- `LLM_API_KEY` / `SUPABASE_SERVICE_ROLE_KEY` 一旦泄漏到前端代码或 GitHub，会被恶意滥用，账单可能爆掉
- 公开上线后，恶意用户可能用脚本疯狂请求 `/api/chat`，单日烧光预算

**必须做**：

- [ ] 检查所有 API 密钥**只**在 `app/api/*` 服务端代码中使用，**绝不**出现在 `app/(client)/*` 或 `components/`
- [ ] 用 GitHub 的 secret scanning 检查仓库历史中是否有密钥泄漏
- [ ] 在 `app/api/chat` 等高消耗端点加 rate limit（按 IP 限制：每分钟 ≤ 10 次，每天 ≤ 200 次）
- [ ] DeepSeek / Anthropic Console 里设置月度 spending limit（如 $100 / 月硬上限，触达后停服务）
- [ ] Supabase 启用 RLS（Row Level Security），确保用户 A 无法查到用户 B 的数据

## 4. 费用预算与监控

**估算每用户每月成本**：

| 项目 | 量 | 单价 | 月成本 |
|------|---|------|-------|
| LLM 调用（DeepSeek） | ~30 节课 × 20 轮对话 × 2K token = 1.2M token | ¥2/M | ~¥3 |
| Supabase 存储 | 数据库 < 1MB / 用户 | 免费额度 | ¥0 |
| Vercel 流量 | < 100MB / 用户 | 免费额度 | ¥0 |
| **合计** | | | **~¥3 / 用户 / 月** |

**预算决策**：上线前确定可承受的月度上限。比如个人付费支撑 100 用户 = ¥300/月。

**监控**：

- [ ] DeepSeek / Anthropic 后台设月度账单告警（达到 50% / 80% / 100% 触发邮件）
- [ ] 在产品里加一个简单的 `/admin` 页面，能看到当天调用次数、活跃用户数（不需要复杂，console.log 到 Vercel Logs 也行）

## 5. AI 内容免责声明 UI 落地

PRD.md §15.4 已写「热点摘要仅用于学习辅助，重要信息以原文为准」。但**所有 AI 生成内容都需要类似声明**。

**必须做**：

- [ ] 三导师对话气泡底部加小字：「以上由 AI 生成，仅供学习参考」
- [ ] 论文导读类课程加："论文要点仅为学习辅助，建议阅读原文"
- [ ] 用户上传资料拆课结果加："AI 生成的课程结构，可能与原文有偏差"

## 7. 域名、备案与合规（如果面向中国大陆用户）

**如果只面向海外用户**：买个 .com / .ai 域名挂 Vercel 即可。跳过本节。

**如果面向中国大陆用户**：

- [ ] 注册中国大陆服务器商（阿里云 / 腾讯云）
- [ ] ICP 备案（个人或公司主体）—— 1-2 个月
- [ ] 内容自查：AI 生成内容如果涉及时政、新闻、医疗、金融，需要更严的合规审查
- [ ] 数据出境合规：用 DeepSeek 国内 API、Supabase 部署在国内/香港，避免数据跨境

**决策动作**：先决定面向用户群，再选合规路径。
